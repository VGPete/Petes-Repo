  • Form Validation
    • Client side form validation allows users to fix issues before they get to the server, for security and to avoid incorrect data or delays.
    • Validation done in the browser is called client side validation, but if done in the server its server side validation.
    • The two types of validation are 1. Built-in & 2. Custom JavaScript
      • Built-in: Includes options to validate forms such as Required, minlength/maxlength, min/max for numerical values, type (email, number, etc), and pattern which allows for custom regex.
      • JavaScript Allows for the constraint validation API which makes it very easy to validate forms. You can however still do completely custom validation without the API.
  • Fetch API
    • The fetch API allows easy access to requests from server side content.
    • It returns a promise which if everything goes smoothly will provide the content requested.
    • The init object allows you to manipulate many settings of the fetch request including cors info, content type, credentials, method etc.
    • Fetch can be used to do many things like access data on a server, upload data or files to a server.
    • Fetch has a catch feature for errors
